# Как изменить URL страницы на .pages.dev

Есть два способа изменить адрес вашего сайта на Cloudflare Pages:

---

## Вариант 1: Изменить имя проекта (новый .pages.dev адрес)

Если вы хотите изменить URL с `https://iqtest.pages.dev` на другой (например, `https://iq-test.pages.dev`):

### Шаг 1: Переименовать проект

1. Откройте: https://dash.cloudflare.com/pages
2. Найдите ваш проект `iqtest`
3. Откройте проект
4. Перейдите в **Settings** → **General**
5. Найдите раздел **Project name**
6. Нажмите **Edit** (или иконку карандаша)
7. Введите новое имя (например: `iq-test`, `iqtest-online`, `quick-iq-test`)
8. Нажмите **Save**

**Важно:** 
- Имя должно быть уникальным (не занято другим проектом)
- Может содержать только буквы, цифры и дефисы
- После переименования URL изменится автоматически

### Шаг 2: Новый URL

После переименования сайт будет доступен по новому адресу:
```
https://новое-имя.pages.dev
```

**Примеры:**
- `https://iq-test.pages.dev`
- `https://iqtest-online.pages.dev`
- `https://quick-iq-test.pages.dev`

---

## Вариант 2: Добавить кастомный домен

Если у вас есть свой домен (например, `iqtest.com`), вы можете использовать его вместо `.pages.dev`:

### Шаг 1: Добавить кастомный домен

1. Откройте проект в Cloudflare Pages
2. Перейдите в **Settings** → **Custom domains**
3. Нажмите **"Set up a custom domain"**
4. Введите ваш домен (например: `iqtest.com` или `www.iqtest.com`)
5. Нажмите **Continue**

### Шаг 2: Настроить DNS записи

Cloudflare покажет, какие DNS записи нужно добавить:

**Обычно нужно добавить:**
- **CNAME запись:**
  - Name: `@` (или `www` для поддомена)
  - Target: `iqtest.pages.dev` (ваш текущий .pages.dev адрес)
  - Proxy: включен (оранжевое облачко)

**Где добавить:**
- Если домен в Cloudflare: DNS → Records → Add record
- Если домен в другом регистраторе: добавьте CNAME в панели управления доменом

### Шаг 3: Дождаться обновления DNS

- Обычно 5-30 минут
- Можно проверить статус в настройках Custom domains

### Шаг 4: Проверить

После обновления DNS сайт будет доступен по вашему домену:
```
https://iqtest.com
```

**Важно:** `.pages.dev` адрес продолжит работать параллельно с кастомным доменом.

---

## Вариант 3: Создать новый проект с нужным именем

Если переименование не работает или вы хотите начать заново:

1. Создайте новый проект в Cloudflare Pages
2. При создании укажите нужное имя проекта
3. Подключите тот же GitHub репозиторий
4. Настройки:
   - Build output directory: `public`
   - Build command: пустой
5. После деплоя старый проект можно удалить

---

## Как узнать текущий URL проекта

1. Откройте проект в Cloudflare Pages
2. В верхней части страницы вы увидите:
   - **Production URL:** `https://iqtest.pages.dev`
   - Или другой адрес, если был добавлен кастомный домен

## Часто задаваемые вопросы

### Можно ли изменить имя проекта после создания?

**Да**, но:
- Нужно зайти в Settings → General → Project name → Edit
- Новое имя должно быть уникальным
- URL изменится автоматически

### Можно ли использовать несколько доменов?

**Да**, можно добавить несколько кастомных доменов:
- `iqtest.com`
- `www.iqtest.com`
- `test.iqtest.com`
- И т.д.

Все они будут указывать на один и тот же сайт.

### Что делать, если имя проекта занято?

Выберите другое имя:
- `iq-test-online`
- `iqtest-app`
- `quick-iq-test`
- `iqtest-ru`
- И т.д.

### Можно ли удалить .pages.dev адрес?

**Нет**, `.pages.dev` адрес всегда остается активным, даже если добавлен кастомный домен. Это нормально и не мешает работе.

---

## Резюме

**Самый простой способ изменить URL:**
1. Откройте проект в Cloudflare Pages
2. Settings → General → Project name → Edit
3. Введите новое имя
4. Сохраните
5. URL изменится автоматически

**Для кастомного домена:**
1. Settings → Custom domains → Set up a custom domain
2. Добавьте DNS записи
3. Дождитесь обновления DNS
